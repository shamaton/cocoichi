## このドキュメントの位置づけ

この文書は、`docs/poc-home-tab-architecture-2026-04-12.md` を画面レベルまで具体化するワイヤー資料である。
`S1-S8` の注文主導線そのものを置き換えるものではなく、`Home` `Menu` `Order` `Tab Bar` の情報優先度と状態差を補完する。

## 目的

この資料は、PoC の `アプリ起動直後の入口` と `タブバー導線` を具体化するためのワイヤーです。
見た目の最終デザインではなく、`初回起動時に何を見せるか` `店舗未設定と設定済みで何を変えるか` `注文導線へどう接続するか` を定義します。

## 前提

- 端末は iPhone を想定
- 起動直後の初期表示は `Home` タブ
- `Home` と `Menu` は店舗未設定でも閲覧可能
- 注文開始や店舗変更時に `S1 Store Select` を開く
- 店舗限定メニューは `店舗設定済み` の時だけ表示する
- タブバーは Liquid Glass 系の軽い素材を許容するが、可読性を優先する

## 画面の役割整理

### Home

- 起動直後の受け皿
- 期間限定、おすすめの発見
- `受取先を選ぶ` の強い入口
- 他タブへのショートカット

### Menu

- フルのメニュー探索
- 検索、絞り込み、定番、おすすめから商品を探す
- 店舗設定済みなら店舗限定メニューも混ぜる

### Order

- 現在の注文状態へ最短で戻る
- 未設定時は `注文を始める` の空状態を見せる

### Tab Bar

- 全画面共通のベース導線
- 主要機能への移動を最短化する
- 注文主導線を切らず、今いる場所を分かりやすく示す

### Home コンポーネント意図

- `Fulfillment Card`
  - 最上位に置き、店舗未設定でも次に何をすべきかを明確にする
- `Seasonal Banner`
  - 販促ではなく食欲を立てる導線として使う
- `This Store Only`
  - 店舗設定済み時だけ出し、限定メニューの価値を伝える
- `Recommended`
  - 初回でも迷わず触れる入口にする

### Home 重要ルール

- 店舗未設定時でも `空画面` にしない
- 他タブへの案内はホームを食わない補助導線に留める
- `オーダーを始める` は店舗未設定なら `S1 Store Select` を開く
- 店舗限定メニューは店舗設定済み時だけ見せる

### Menu コンポーネント意図

- `Store Prompt Card`
  - 店舗未設定でも閲覧を止めず、限定情報だけ後から解放する
- `Store Header`
  - 設定済み時は店舗と受取目安を短く固定表示する
- `Genre Header`
  - `カレー / サラダ / ドリンク / その他` を横並びで見せ、幅が狭い時は横スクロールで辿れるようにする
  - PoC 初期段階では `カレー` 以外はプレースホルダーでもよい
- `This Store Only`
  - 限定メニューに気づかせる最短導線
- `Menu List`
  - 一覧導線は S2 の情報構造に自然につながるように保つ

### Menu 重要ルール

- 店舗未設定時もメニュー閲覧は止めない
- 商品タップ後に注文導線へ入る場合は `S1` を挟んでよい
- 店舗未設定で商品タップ後に `S1` を挟む場合は、店舗確定後に `S2` を戻り履歴へ積み、表示はスキップして `S3` に進める
- `S3` から戻ると店舗文脈つきの `S2`、さらに戻ると `S1` に戻る
- 店舗限定メニューは `別タブ` にしない
- `限定` は価値訴求であり、主役はあくまでカレー選択体験
- `カレー` をデフォルトにし、非カレーは横スワイプで見つけられる補助導線として置く

### 情報構造

1. 現在の注文状態
2. 受取先コンテキスト
3. 次にやること

### Order 重要ルール

- `Order` は注文の再開場所であり、販促面にしない
- 店舗未設定時は `始め方` を明確に示す
- draft や cart がある時は、この空状態を出さず `S5 Order Review` へ戻す

## Tab Bar

### ワイヤー

#### 通常状態

```text
┌─────────────────────────────────────┐
│                                     │
│         各タブのコンテンツ領域         │
│                                     │
├─────────────────────────────────────┤
│   Home    Menu    Order   Rewards   │
│  [house] [fork]   [bag]   [star]    │
└─────────────────────────────────────┘
```

#### Menu root / 確定済み商品あり

```text
┌─────────────────────────────────────┐
│                                     │
│         Menu コンテンツ領域           │
│                                     │
├─────────────────────────────────────┤
│              [cart ●2]  1,980円      │
└─────────────────────────────────────┘
```

### タブバー意図

- ラベルは短く、意味を誤読しにくい単語にする
- アイコンは SF Symbols 基準で線ベースを中心にする
- 現在タブは塗りまたは暖色アクセントで示す
- 背景は軽い Liquid Glass を許容するが、写真の上でも文字が埋もれない濃度にする

### タブバー重要ルール

- `クーポン` は独立タブにしない
- `Rewards` は将来機能の置き場として残すが、未実装感を強く出しすぎない
- 選択状態は色だけに依存せず、アイコンの塗りやラベルウェイト差でも示す
- ガラス感は演出ではなく階層整理のために使う
- `Menu` root で確定済み商品が1点以上ある場合は、通常のタブバーを一時的に隠し、カートアイコン、注文数バッジ、合計金額だけのフッターに差し替える
- カートフッター押下時は `S5 Order Review` に戻し、未確定の商品探索より現在の注文確認を優先する

## 状態遷移メモ

- `Home -> 受取先を選ぶ` で `S1 Store Select`
- `Home -> 期間限定バナー` で `Menu` または `S2`
- `Menu -> 商品タップ` で
  - 店舗設定済みなら `S3`
  - 店舗未設定なら `S1` を経由し、`S2` を履歴に積んで `S3`
- `S3 -> 戻る` で `S2`、さらに `戻る` で `S1`
- `Menu root + cart line item あり -> カートフッター` で `S5 Order Review`
- `Order -> 受取先を選んで注文を始める` で `S1`
- `Order -> メニューを見る` で `Menu` または `S2`

## この資料が固定すること

- 起動直後は `Home` を見せる
- `Home` と `Menu` は店舗未設定でも成立させる
- 店舗設定済み時だけ `店舗限定メニュー` を出す
- 注文主導線に入る瞬間に `S1` を使う
- 店舗未設定で先に商品を選んだ場合は、`S1 -> S2 -> S3` の戻り履歴を作る
- タブバーは `Home / Menu / Order / Rewards` の 4 つで整理する
- ただし `Menu` root で確定済み商品がある間は、タブバーより注文確認へのカートフッターを優先する
