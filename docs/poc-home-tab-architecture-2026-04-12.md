競合がある場合は次を優先する。

補完関係:

- `docs/poc-screen-flow-2026-03-28.md` の `S1 -> S2 -> S3 -> S5 -> S6 -> S8` は `注文開始後の主導線` として維持する
- 本ドキュメントでは、`アプリ起動直後に必ず S1 を強制表示するか` ではなく、`ホーム閲覧と注文開始をどう両立するか` を定義する

## 背景

PoC の主導線は `まず店舗を選ぶ` を前提にしている。
一方で、インストール直後のユーザーは `まずメニューを見たい` `限定メニューやおすすめを見てから決めたい` という行動を取りやすい。

このため、次の 2 つを同時に満たす必要がある。

1. 注文開始後は、店舗コンテキストが常に見えること
2. 注文前の閲覧までは、店舗設定を必須にしないこと

## 設計結論

### 1. アプリ起動直後は `Home` タブを初期表示にする

- 初回起動でいきなり `S1 Store Select` を強制しない
- まずは `おすすめ` `期間限定` を軽く見せる
- `受取先を選ぶ` はホーム上部の強い導線として常時見せる

### 2. `S1 Store Select` は `注文開始ゲート` として扱う

- `オーダーを始める`
- `クーポンを使う`
- `店舗を変更する`
- `店舗限定メニューを注文に進める`

上記の瞬間に `S1` を開き、店舗または受取条件を確定させる。

### 3. 閲覧は店舗未設定でも許可する

- `Home` と `Menu` は店舗未設定でも閲覧可能
- ただし店舗未設定時は `受取時間` `店舗限定メニュー` `店舗別可否` は確定表示しない
- 注文確定やクーポン適用など、店舗文脈が必要な操作では `S1` を挟む

### 3-a. 商品選択後の店舗ゲートは戻り履歴に残す

店舗未設定の `Menu` で商品を選んだ場合は、`S1 Store Select` を挟んで店舗を確定する。
店舗確定後は `S2 Menu Discovery` を NavigationStack 上に積み、その表示はスキップして、選択済み商品の `S3 Curry Detail / Customize` へ直接進める。

```text
Home
  -> Menu
  -> 商品選択
  -> S1 Store Select
  -> S2 Menu Discovery を stack
  -> S3 Curry Detail / Customize
```

戻る操作は次を基準にする。

```text
S3 Curry Detail / Customize
  -> S2 Menu Discovery
  -> S1 Store Select
```

このため、店舗選択は `いつでも自由に差し替えるグローバル設定` ではなく、`注文可能な状態に入るためのゲート` として扱う。
注文作成中の店舗変更は、戻る操作または明示的な変更操作に寄せ、既存注文内容がある場合は破棄確認を必須にする。

### 4. クーポンはタブにしない

- PoC では `クーポン専用タブ` を置かない
- クーポンは `Home` で軽く予告し、実適用は `S5/S6` の文脈で見せる
- `得させるために主導線を曲げる` のではなく、`注文文脈の中で自然に提案する` を守る

## タブ構成

PoC 初版のタブは次を基本とする。

1. `ホーム`
2. `メニュー`
3. `オーダー`
4. `リワード`

### タブ 1: ホーム

役割:

- アプリ起動直後の受け皿
- 他タブへの案内
- 季節メニューやおすすめの発見
- `受取先を選ぶ` への入口

主要モジュール:

1. `受取先カード`
2. `期間限定メニューバナー`
3. `おすすめメニュー`
4. `他タブショートカット`

状態別の見せ方:

- 店舗未設定時
  - ヘッダーに `受取先を選ぶと限定メニューも見られます` を表示
  - メニューは共通カタログのみ見せる
- 店舗設定済み時
  - `渋谷道玄坂店 / 12-18分` のような店舗カードを固定表示
  - その店舗で推せるメニューや限定商品を上位に混ぜる

### タブ 2: メニュー

役割:

- フルのメニュー探索
- 検索、絞り込み、定番、おすすめ、Saved Combos からの流入
- 注文導線の手前にある発見画面

状態別の見せ方:

- 店舗未設定時
  - 共通メニューのみ表示
  - 上部に `受取先を選ぶとこの店舗限定メニューも表示` の案内カードを置く
  - 商品タップ後に注文へ進む場合は `S1` を開く
- 店舗設定済み時
  - 共通メニューに加え、`この店舗限定` セクションまたは限定バッジを表示する
  - 店舗限定商品は一覧に自然に混ぜてよいが、最上部に 1 セクション持つ方が気づきやすい

### タブ 3: オーダー

役割:

- 現在の注文状態へ最短で戻る
- `pending draft` `cart` `直前の受取先` を確認する

状態別の見せ方:

- 店舗未設定かつ draft なし
  - `受取先を選んで注文を始める` を主CTAで見せる
  - CTA で `S1` を開く
- 店舗設定済みで draft なし
  - `メニューを見る` から `S2` に入れる
- draft または cart あり
  - 直接 `S5 Order Review` へ戻せる

### タブ 4: リワード

役割:

- 将来のスタンプ / 会員価値の置き場
- PoC 初版では `将来機能のプレースホルダー` に留める

制約:

- 注文主導線の邪魔をしない
- ログイン要求を前面に出さない
- 初版では実機能より `将来の情報設計の余白` として扱う

## 受取コンテキスト

PoC 全体で、`店舗選択` を `受取コンテキスト` として扱う。

受取コンテキスト状態:

1. `未設定`
2. `店舗受取`
3. `デリバリー想定`

PoC 初版では `店舗受取` を主対象にする。
`デリバリー想定` はタブ導線上の入口だけ持てばよく、注文成立の細部は広げすぎない。

ルール:

- `受取コンテキスト` は全タブ共通のグローバル状態として持つ
- 設定済みなら、ホーム / メニュー / オーダーで常に短い要約を見せる
- 受取コンテキスト変更時は、既存仕様通り `cartItems / pending draft / applied coupon` を破棄する

## 店舗限定メニューの扱い

### 基本ルール

- 共通メニューは店舗未設定でも閲覧できる
- 店舗限定メニューは `店舗設定済み` の時だけ一覧に出す
- 店舗限定メニューを閲覧可能にするトリガーは `選択中の店舗` とする

### UIルール

- `Home`
  - 店舗設定済み時のみ `この店舗の限定メニュー` を出す
- `Menu`
  - `この店舗限定` セクションを最上部近くに出すか、商品カードに `限定` バッジを付ける
- `S2 Menu Discovery`
  - 現在の店舗ヘッダー直下に `この店舗で選べる限定メニュー` を混ぜてもよい
  - ただし主役はあくまで `ベースのカレーを選ぶ` 体験であり、販促棚のようにしない

### データルール

メニューの表示可否は少なくとも次で判定できる構造にする。

- `isGlobal`
- `availableStoreIDs`
- `availabilityNote`

初版 PoC では:

- 店舗限定商品がまだ存在しなくても、モデル上は対応可能にしておく
- 限定商品が 0 件の店舗では、UI を無理に出さない

## 初回起動の想定フロー

### 1. まず見たい人

```text
App Launch
  -> Home
  -> 期間限定やおすすめを見る
  -> Menu
  -> 商品をタップ
  -> S1 Store Select
  -> S2 Menu Discovery を stack
  -> S3 Curry Detail / Customize へ進む
```

### 2. 店舗で今すぐ注文したい人

```text
App Launch
  -> Home
  -> 受取先カードをタップ
  -> S1 Store Select
  -> S2 Menu Discovery
  -> S3
  -> S5
```

### 3. 店舗設定済みで再訪した人

```text
App Launch
  -> Home
  -> 現在の店舗カードと限定メニューを見る
  -> おすすめから注文を再開
```

## 実装示唆

### App Shell

- `TabView` を app の最上位骨格に置く
- 各タブの中に `NavigationStack` を持つ
- `S1` は通常の受取先設定ではフルスクリーン遷移または sheet で呼び出せるようにする
- 店舗未設定の商品選択後に開く `S1` は、戻り履歴を成立させるため NavigationStack 上の screen としても扱えるようにする
- 店舗選択後に選択済み商品へ直行する場合は、`S2` を path に含めてから `S3` を表示する

### 状態管理

- `selectedFulfillmentMode`
- `selectedStore`
- `visibleMenuCatalog`
- `storeSpecificMenuItems`
- `hasStoreScopedContent`

を持てる構造にする。

### Menu Discovery

- `S2` のベース一覧は `selectedStore` の有無で切り替える
- 店舗未設定時は共通メニューだけを返す
- 店舗設定済み時は `共通 + 店舗限定` を返す
- 店舗未設定時の商品タップでは、選択中の商品を一時保持し、S1 完了後に同じ商品で S3 を開始する
- S3 から戻った時に表示される S2 は、選択済み店舗の文脈を持つメニューとして表示する

## 非採用方針

- 初回起動でいきなり店舗選択を強制する
- 注文作成中に店舗をヘッダーから即時自由変更できるようにする
- クーポン専用タブを作る
- 店舗限定メニューを独立タブに切り出す
- リワード導線で注文主導線を塞ぐ
- 店舗変更後も旧店舗の draft や cart を温存する

## このドキュメントが補う論点

- `アプリ全体のホーム` と `注文フローの S1/S2` の役割分担
- `店舗未設定でも見たい` という初回ユーザー要求への対応
- `店舗設定済みなら限定メニューも混ぜる` という将来拡張の受け皿
