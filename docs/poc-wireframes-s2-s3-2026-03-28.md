## 目的

この資料は、PoCの中心になる `商品を探す` 体験と `注文を作る` 体験を具体化するためのワイヤーです。
見た目の最終デザインではなく、`情報の優先順位` `操作の流れ` `主CTAの位置` `状態変化` を定義します。

## 前提

- 端末は iPhone を想定
- iOS-first
- 主導線は片手操作を前提に、主要CTAを画面下部へ寄せる
- `検索` `保存` `確認へ進む` の3操作が迷子にならないことを優先する
- `S2` は店舗未設定でも閲覧できる Menu Discovery と、`S1` で店舗確定後に入る店舗コンテキスト付き discovery の両方を扱う
- 店舗未設定の S2 で商品を選んだ場合は、`S1` を挟み、店舗確定後に `S2` を履歴へ積んだまま `S3` へ直接進める
- 店舗設定済み時は、共通メニューに加えて店舗限定メニューを自然に混ぜる

## S2 Menu Discovery

### 画面の役割

- 今の気分から商品を選ぶ
- 定番とおすすめを自然に見つけさせる
- Saved Combos や検索にすぐ移れる

### 情報構造

1. navigation 上の店舗コンテキスト
2. 検索と軽い絞り込み
3. 店舗限定または店舗固有の提案
4. 発見導線
5. 商品一覧
6. Saved Combos と探索補助導線

### S2 コンポーネント意図

- `Navigation Store Title`
  - 店舗がどこかをジャンル選択 UI より上の navigation 領域に見せる
  - 店舗名タップで履歴上の `S1 Store Select` に戻れるようにする
  - メニュー本文内には店舗情報カードを置かない
- `This Store Only`
  - 店舗限定メニューや、その店舗で強く推したい商品に気づかせる
  - 1 セクションで十分とし、販促棚のように増やしすぎない
- `Search Field`
  - 商品名、トッピング名、気分ワード検索を想定
- `Quick Filters`
  - タップ時に軽い反応を返す小さなチップ
- `For You`
  - Saved Combos と現在の気分をつなぐ再利用導線
- `Popular Today`
  - 写真起点で直感的に選ばせる
- `Menu List`
  - 最終的な一覧導線
- `Bottom Action Area`
  - `Saved Combos` と探索補助の入口を固定

### S2 重要ルール

- 商品一覧に行く前に検索やおすすめから入れてよい
- Saved Combos は独立機能だが、S2 内からも常に見える
- 1タップで `S3 Curry Detail / Customize` に進める
- 店舗未設定時の商品タップは `S1 Store Select` を挟み、店舗確定後に同じ商品で `S3` を開始する
- その場合、`S3` から戻ると店舗文脈つきの `S2` が表示され、さらに戻ると `S1` に戻る
- 店舗限定メニューは店舗ヘッダー直下または一覧内バッジで見せ、独立タブ化しない
- 限定商品が 0 件の店舗では、無理にプレースホルダーを出さない
- `限定` は強い補助導線だが、S2 全体を販促面にしない
- Home や Menu から入ってきた時も、S2 では `店舗が決まった discovery 画面` に切り替わることを明確に見せる
- 店舗設定済みの注文文脈内では、navigation の店舗名から履歴上の S1 に戻す。即時自由変更にはしない
- `続けて注文` で S5 から入った S2 は、店舗名をジャンル選択 UI の上に表示し、戻る操作で S5 に戻れるようにする

## S3 Curry Detail / Customize

### 画面の役割

- 商品詳細を見せる
- 基本設定を一画面でまとめ、トッピングを次 phase で調整させる
- 選択結果の価格と見た目を即時反映する
- 編集と確認遷移を迷わせない

### 情報構造

1. 商品の魅力と現在価格
2. phase 切り替えと現在の進捗
3. 現在構成のサマリー
4. 基本設定またはトッピングのカスタマイズ内容
5. 次 phase / Review CTA

### S3 手順整理

`S2 Menu Discovery` でベースのカレーを選んだ後、`S3 Curry Detail / Customize` は次の順で進める。
各 phase で表示する具体要素は後続で詰める。

1. 基本設定 phase で `カレーソース` `ライス量` `辛さ` を一画面で決める
2. `ソース量` は折りたたみで必要時だけ調整する
3. トッピング phase で `追加トッピング` を決める
4. `Review Order` で S5 に進む

補足:

- 画面上部または下部に `基本設定 / トッピング` の phase 表示を置き、現在地と残り工程を短く伝える
- 以下のワイヤーは `基本設定 phase` と `トッピング phase` の優先情報を定義するものであり、装飾は後続で調整してよい
- PoC 初版では `一度に全部決めさせる` ではなく、`基本設定は速く、トッピングは迷わず選べる` を優先する
- 店舗未設定の S2 から商品を選んで入った場合、`戻る` は店舗文脈つき S2 へ戻し、さらに戻ると S1 で店舗を選び直せる
- S3 以降では店舗を直接自由変更させず、変更が必要な時は戻る履歴または確認付き操作に寄せる

### S3 コンポーネント意図

- `Hero Image`
  - 食欲を作る
  - 選択変化に対して微小な反応を返す
- `Phase Switcher`
  - `基本設定 / トッピング` の現在地を短く伝える
  - 次に何をするかを迷わせない
- `Order Snapshot Card`
  - 今何を作っているかを常に短く把握させる
- `Store Only Badge`
  - 限定商品から入った時だけ Hero 付近または Snapshot 内に短く残す
  - カスタマイズ操作の主役を奪わない
- `Basics Content Area`
  - `カレーソース / ライス量 / 辛さ` を一画面でまとめて決められるようにする
- `Sauce Amount Disclosure`
  - 必要な人だけが開ける補助調整として扱う
- `Bottom CTA Area`
  - `次 phase / Review` を迷わず進められるようにする
- `Footer Selection Summary`
  - 選択済みトッピングと個数を下部で安定して確認できるようにする
  - 個数調整後も主CTAの位置を見失わせない

### S3 重要ルール

- 価格は必ずリアルタイム更新
- phase 表示で `今どこか` が常に分かる
- 順番は `基本設定 -> トッピング` を崩さず、`Review Order` CTA から S5 へ抜ける
- 基本設定 phase の中では `カレーソース -> ライス量 -> 辛さ` を視認しやすく並べ、`ソース量` は折りたたみで補助的に扱う
- `Order Snapshot Card` は S3 の両 phase で常に見える位置を保ち、店舗文脈を落とさない
- 限定商品から入った時は `この店舗限定` を短く残すが、主CTAより目立たせない
- 選択変化は、数値だけでなく視覚的にも反応する
- 保存導線は完了後の `S8` に寄せ、S3 では注文を作ることを優先する
- `次へ進む / Review` は迷いなく押せる主CTAとして扱う
- トッピングは同一商品を複数個選べるようにし、候補カード内で `追加 / - / +` の数量調整を完結させる
- 選択済みトッピングと個数は画面上部へ積まず、下部の summary で確認できるようにする
- Review へ進む前にログインを要求しない

## 遷移メモ

```text
S2 Menu Discovery
  -> tap menu card
  -> S3 Curry Detail / Customize

S3 Curry Detail / Customize
  -> tap Review Order
  -> S5 Order Review
```

## アニメーションと反応の最低要件

- S2 で商品カードを押したときに、S3 の Hero Image へ滑らかにつながる
- S3 でトッピング追加時に、Order Snapshot Card と価格に即時反応がある
- 反応は速く、0.2秒から0.45秒程度の短い体感を目安にする

## アクセシビリティ注意点

- 商品名、価格、トッピング追加状態を VoiceOver で読める構造にする
- 色だけで選択状態を示さない
- 主要CTAのタップ領域を十分に確保する
- Hero Image の演出があっても情報の読みにくさを起こさない

## 次に詰める論点

1. S2 の `This Store Only` を常に先頭固定にするか、商品一覧内のバッジ中心に寄せるか
2. S2 の `For You` を限定セクションの後に置くか、検索下に抑えるか
3. S2 の商品一覧をカード中心にするか、リスト中心にするか
4. S3 で `この店舗限定` を Hero 側に置くか Snapshot 側に寄せるか
5. S3 のトッピングを grid にするか、カテゴリ別 horizontal scroll にするか
