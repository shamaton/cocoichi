# CoCo壱番屋 新規アプリ PoC ワイヤー

- 作成日: 2026-03-28
- 対象画面:
  - `S2 Menu Discovery`
  - `S3 Curry Detail / Customize`
- 関連資料:
  - `docs/poc-app-direction-2026-03-28.md`
  - `docs/poc-screen-flow-2026-03-28.md`

## 目的

この資料は、PoCの中心になる `商品を探す` 体験と `注文を作る` 体験を具体化するためのワイヤーです。
見た目の最終デザインではなく、`情報の優先順位` `操作の流れ` `主CTAの位置` `状態変化` を定義します。

## 前提

- 端末は iPhone を想定
- iOS-first
- 主導線は片手操作を前提に、主要CTAを画面下部へ寄せる
- `検索` `保存` `確認へ進む` の3操作が迷子にならないことを優先する

## S2 Menu Discovery

### 画面の役割

- 今の気分から商品を選ぶ
- 定番とおすすめを自然に見つけさせる
- Saved Combos や検索にすぐ移れる

### 情報構造

1. 現在の店舗コンテキスト
2. 検索と軽い絞り込み
3. 発見導線
4. 商品一覧
5. Saved Combos と探索補助導線

### S2 コンポーネント意図

- `Store Context Header`
  - 店舗がどこか、受取目安が何分かを常時見せる
- `Search Field`
  - 商品名、トッピング名、気分ワード検索を想定
- `Quick Filters`
  - タップ時に軽い反応を返す小さなチップ
- `For You`
  - Saved Combos と現在の気分をつなぐ再利用導線
- `Popular Today`
  - 写真起点で直感的に選ばせる
- `Menu List`
  - 最終的な一覧導線
- `Bottom Action Area`
  - `Saved Combos` と探索補助の入口を固定

## S3 Curry Detail / Customize

### 画面の役割

- 商品詳細を見せる
- トッピング、辛さ、量をその場で調整させる
- 選択結果の価格と見た目を即時反映する
- お気に入り保存と確認遷移を迷わせない

### 情報構造

1. 商品の魅力と現在価格
2. 現在構成のサマリー
3. カスタマイズ群
4. お気に入り保存
5. 確認へ進むCTA

### S3 コンポーネント意図

- `Hero Image`
  - 食欲を作る
  - 選択変化に対して微小な反応を返す
- `Current Order Card`
  - 今何を作っているかを常に短く把握させる
- `Spice / Rice Selectors`
  - 迷わせない即時選択UI
- `Toppings Grid`
  - 選び足す楽しさを作る
- `Selected Toppings`
  - 現在構成の確認と削除を簡単にする
- `Bottom CTA Area`
  - 保存と確認の2つだけを並べる

### S3 重要ルール

- 価格は必ずリアルタイム更新
- 選択変化は、数値だけでなく視覚的にも反応する
- `Save Combo` は補助CTA、`Review Order` は主CTA
- Review へ進む前にログインを要求しない

## 遷移メモ

```text
S2 Menu Discovery
  -> tap menu card
  -> S3 Curry Detail / Customize

S3 Curry Detail / Customize
  -> tap Save Combo
  -> Save Favorite Sheet
  -> back to S3

S3 Curry Detail / Customize
  -> tap Review Order
  -> S5 Order Review
```

## アニメーションと反応の最低要件

- S2 で商品カードを押したときに、S3 の Hero Image へ滑らかにつながる
- S3 でトッピング追加時に、Current Order Card と価格に即時反応がある
- Save Favorite 完了時は小さな成功フィードバックを返す
- 反応は速く、0.2秒から0.45秒程度の短い体感を目安にする

## アクセシビリティ注意点

- 商品名、価格、トッピング追加状態を VoiceOver で読める構造にする
- 色だけで選択状態を示さない
- 主要CTAのタップ領域を十分に確保する
- Hero Image の演出があっても情報の読みにくさを起こさない

## 次に詰める論点

1. S2 の `For You` を最上段に置くか、検索下に抑えるか
2. S2 の商品一覧をカード中心にするか、リスト中心にするか
3. S3 の `Current Order Card` を固定表示にするか、スクロール連動にするか
4. S3 のトッピングを grid にするか、カテゴリ別 horizontal scroll にするか
5. Save Combo を右上アイコン起点に寄せるか、下部CTAに残すか
