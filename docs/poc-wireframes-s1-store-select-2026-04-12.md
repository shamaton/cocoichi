## このドキュメントの位置づけ

この文書は、`S1 Welcome / Store Select` を画面レベルまで具体化するワイヤー資料である。
`docs/poc-screen-flow-2026-03-28.md` の遷移定義と `docs/poc-home-tab-architecture-2026-04-12.md` のゲート設計を補完し、`店舗をどう選ばせるか` `どの入口を優先するか` `確定後にどう次へ送るか` を定義する。

## 目的

この資料は、PoC の注文開始ゲートである `S1 Store Select` を具体化するためのワイヤーです。
見た目の最終デザインではなく、`初回でも詰まらないこと` `店舗変更時も軽くやり直せること` `店舗決定後に自然にメニューへ進めること` を優先します。

## 前提

- `S1` はアプリ起動直後の強制画面ではなく、注文開始時または店舗変更時に開く
- 現在地検索だけに依存しない
- `現在地 / 駅名 / 郵便番号 / 店名` の複数入口を用意する
- GPS 失敗時でも手入力検索で先に進める
- 会員設定やお気に入り店舗登録とは分離する
- 店舗確定後は `S2 Menu Discovery` に進む
- `保存済みから始める` は `S4 Saved Combos` への副導線として持つ
- PoC 初版では `店舗受取` を主対象とし、`デリバリー` は入口だけ持つ

## S1 の画面役割

- 注文開始の心理的ハードルを下げる
- 受取コンテキストを明確にする
- 店舗選択を短時間で完了させる
- 決定後すぐにメニュー検討へ移す

## 情報構造

1. 画面の目的と現在モード
2. 受取モード切り替え
3. 検索入口
4. 近くの候補または検索結果
5. 選択後の次アクション
6. `保存済みから始める` の副導線

## S1 コンポーネント意図

- `Mode Switch`
  - 店舗受取とデリバリーの入口を一画面で整理する
- `Quick Start`
  - 現在地だけに頼らず、複数入口をすぐ触れる位置に置く
- `Recently Used`
  - 再訪時の最短導線にする
- `Selected Store Card`
  - `ここに決めた` 感覚を明確に返す
- `Next CTA`
  - 店舗確定後は迷わず `S2` へ送る

## S1 重要ルール

- GPS 失敗で詰まらせない
- 現在地検索を最上段に置いてよいが、唯一の入口にしない
- `お気に入り店舗登録` を前提にしない
- 店舗を選んだ後は `設定完了` より `メニューを見に行く` を強く見せる
- `保存済みから始める` は副導線のまま保ち、主導線を食わない
- 店舗変更時は既存の draft / cart / coupon が消えることを必要なら補助文で伝える

## 状態差メモ

- 初回注文
  - 説明文を少し厚めにして不安を減らす
- 再訪時
  - Recently Used を上げて最短再開を優先する
- 店舗変更時
  - `現在の注文内容はリセットされます` の注意文を追加してよい
- デリバリー選択時
  - PoC では配送ロジックを作り込みすぎず、入口整理に留める

## この資料が固定すること

- `S1` はゲート画面であり、起動時の強制画面ではない
- `現在地 / 駅名 / 郵便番号 / 店名` の 4 入口を持つ
- GPS 失敗時も手入力で続行できる
- 店舗確定後は `S2` へ送る導線を最優先にする
- `保存済みから始める` は S4 への副導線として持つ
