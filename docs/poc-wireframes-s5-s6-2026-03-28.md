## 目的

この資料は、注文の完了直前に発生する `不安` と `迷い` を減らすためのワイヤーです。
特に次を具体化します。

- 注文内容の抜け漏れ確認
- クーポン提案の自然な見せ方
- 完了後保存との役割分離
- 注文確定CTAの強さと安心感

## 前提

- `S5 Order Review` は push 画面
- `S6 Coupon Suggestion Sheet` は bottom sheet
- ログイン要求は出さない
- 実決済や実注文APIは使わず、モック完了前提
- 主要CTAは片手操作しやすい下部固定を優先する

## S5 Order Review

### 画面の役割

- 注文内容を一画面で把握させる
- ここで `注文してよい` と確信できる状態を作る
- クーポン提案と注文確定の優先順位を整理する

### 情報構造

1. 受取情報
2. pending draft の確認
3. pending draft を cart に昇格させて追加注文へ戻る footer secondary CTA
4. cart line item と pending draft の見せ分け
5. クーポン提案の入口
6. 価格サマリーと footer の合計金額 / 2CTA

### S5 コンポーネント意図

- `Pickup Card`
  - どこで受け取る注文かを最初に明示する
- `Your Order Card`
  - 初回到達時は pending draft を短く確認させる
  - 追加注文ループ後は read-only の cart line item と pending draft を見分けやすくする
  - 店舗限定商品が含まれる時は、カード内に短い補助ラベルで残す
- `Suggested Savings`
  - クーポンを別機能ではなく、この注文の一部として見せる
- `Price Summary`
  - subtotal / coupon / total の変化を明確にする
- `Bottom CTA Area`
  - ボタン上に `Total` を固定表示する
  - 左に secondary CTA の `続けて注文`、右に primary CTA の `注文を確定` を置く
  - `続けて注文` 押下時に `pending draft` を cart に昇格させて S2 に戻す

### S5 重要ルール

- `注文を確定` が一番目立つ
- `続けて注文` は secondary CTA として常に押し分けられる
- `View Coupon` は提案だが、主導線を邪魔しない
- `Edit` で `S3 Curry Detail / Customize` に戻れるのは pending draft のみ
- 初版 PoC では cart line item は read-only とする
- `続けて注文` を押した時に pending draft を cart に昇格させる
- `続けて注文` の遷移先で `2皿目のカレー` または `サイドメニュー追加` を選べるようにする
- multi-item 状態でも `Suggested Savings` を残し、footer の `続けて注文` / `注文を確定` の責務を変えない
- クーポン未適用でも注文確定できる
- お気に入り保存は `S8 Order Complete` で提案し、S5 では扱わない
- `この店舗限定` は確認情報として残してよいが、金額や注文確定CTAより強く見せない

## S6 Coupon Suggestion Sheet

### 画面の役割

- いまの注文に適用できる候補だけを見せる
- 比較と適用を短時間で終えさせる
- `クーポン読解` ではなく `クーポン選択` の体験にする

### 情報構造

1. この注文に使える理由
2. 適用可能クーポン候補
3. 適用結果の見込み
4. 適用せず閉じる導線

### S6 コンポーネント意図

- `Best Match`
  - 最初に最適解を出す
- `More Options`
  - 適用不可のものは参考情報に留める
- `New Total Preview`
  - 金額メリットを瞬時に理解させる
- `Maybe Later`
  - 適用しない自由を明示する

### S6 重要ルール

- 適用可能な候補を最上位に固定する
- 適用不可クーポンは押せない状態で見せるか、出さない
- 説明文の長文は避ける
- シートを閉じたときは `S5 Order Review` に必ず戻す

## S5 -> S6 の見せ方

### 推奨案

- `S5` で `Suggested Savings` カードを常時表示
- 初回到達時のみ、短い遅延で `S6` を自動ハーフオープン
- ユーザーが閉じたら、その後は CTA 起点に戻す

理由:

- 自然提案の要件を満たしつつ、強制感を下げられる
- 毎回の自動表示を避けられる

### 代替案

- 常に CTA 起点のみ

懸念:

- `自然に提案される` 体験が弱くなる

## 状態差分

### S5 の主要状態

1. クーポン未適用
2. クーポン適用済み

### S6 の主要状態

1. 適用可能クーポンあり
2. 適用直後
3. 適用せず閉じる

## アニメーションと反応の最低要件

- `S5` 到達時、下部CTAはすぐ押せる状態で表示する
- `S6` は bottom sheet が軽く持ち上がるように出る
- クーポン適用時は価格サマリーと CTA 金額が同時更新される
- `注文を確定` は押下時に短い圧縮反応を返す

## アクセシビリティ注意点

- `Applied` `Not Available` を色だけで表現しない
- 合計金額の更新は VoiceOver に通知できる構造にする
- bottom sheet 内の要素密度を上げすぎない

## 次に詰める論点

1. `S6` の自動ハーフオープンを初回のみで十分とするか
2. `Pickup Card` の変更導線を PoC で有効にするか、表示のみとするか
3. 適用不可クーポンを PoC に表示するか、完全に隠すか
4. `注文を確定` 押下後のローディング演出を `S5` で見せるか、即 `S8` に遷移させるか
